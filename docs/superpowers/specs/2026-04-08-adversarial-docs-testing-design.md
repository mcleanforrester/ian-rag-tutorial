# Adversarial Document Testing Suite — Design Spec

**Date:** 2026-04-08  
**Status:** Approved

## Overview

A pytest suite that characterizes how the RAG pipeline handles malformed, edge-case, and adversarial documents. Tests exercise existing `ingestion/` and `retrieval/` components as-is — no code changes, no live server. The goal is to document current failure modes so they can be fixed in a follow-up.

## Approach

Option B: fixture-first, scenario-grouped. Shared fixtures in `conftest.py`, one test file per scenario. Mocked embeddings (deterministic, no Ollama required). Real in-memory ChromaDB (`chromadb.EphemeralClient`). Real binary PDFs generated by `reportlab` at session scope in `tmp_path`.

## Directory Structure

```
tests/
  adversarial/
    conftest.py
    test_scanned.py
    test_long_doc.py
    test_formatting.py
    test_near_dup.py
    test_bad_metadata.py
    test_hallucination.py
```

## Shared Fixtures (`conftest.py`)

- **`mock_embeddings`** — deterministic fake embedding function; hashes text to a fixed-dim float vector. Never calls Ollama.
- **`in_memory_store(docs)`** — builds a `Chroma` instance backed by `chromadb.EphemeralClient()`, indexes given docs, returns a `DocumentRepository`. Torn down after each test.
- **`make_image_pdf(tmp_path_factory)`** — session-scoped; generates a PDF where all content is a JPEG image (no text layer) via `reportlab`. Uses `tmp_path_factory` (not `tmp_path`) to be compatible with session scope.
- **`make_long_pdf(tmp_path_factory, pages=120)`** — session-scoped; generates a 120-page PDF with ~500 words of HR-themed boilerplate per page. Uses `tmp_path_factory`.

## Scenarios & Assertions

### 1. Scanned PDF (`test_scanned.py`)
Documents the silent failure mode of `PyPDFLoader` on image-only PDFs.

- `PyPDFLoader` loads the image PDF without raising an exception
- All loaded docs have empty or whitespace-only `page_content`
- `split_documents()` on those docs returns 0 chunks

### 2. Very Long Document (`test_long_doc.py`)
Documents chunking behavior and retrieval stability on large inputs.

- `split_documents()` on a 120-page PDF completes without error and produces > 500 chunks
- `find_relevant()` returns exactly `k=4` chunks (no explosion or truncation)
- Every returned chunk has a `start_index` metadata key (splitter preserves provenance)

### 3. Inconsistent Formatting (`test_formatting.py`)
Documents that no sanitization occurs — artifacts flow into chunks as-is.

- Chunking does not raise on content mixing raw HTML, Markdown headers, pipe-table syntax, and latin-1 characters in a UTF-8 string
- At least one resulting chunk contains a detectable artifact (HTML tag, `##` header, or `|` table delimiter)

### 4. Near-Duplicate Documents (`test_near_dup.py`)
Documents retrieval capture: whether one duplicate monopolizes the top-k slots.

- Two docs with 95% identical content (one word changed per paragraph), same metadata, both indexed
- `similarity_search` for a matching query returns chunks from **both** documents (not all 4 slots captured by one)
- At most 2 of the 4 retrieved chunks share the same source file

### 5. Missing / Wrong Metadata (`test_bad_metadata.py`)
Documents that malformed metadata causes silent filtering, not crashes.

- `find_relevant()` does not raise when the store contains docs with missing `department` key, `department=None`, or a typo key (`deparment`)
- Those malformed-metadata docs are not returned for a valid user query (they're filtered out, not surfaced)

### 6. Confidently Wrong Context (`test_hallucination.py`)
Documents that the retrieval layer surfaces contradictory facts with no resolution — the attack vector for confident wrong answers.

- Two docs with directly contradictory facts (e.g., "PTO accrual is 15 days/year" vs "25 days/year"), same department/permission
- Both contradictory chunks appear in the top-4 retrieved results for a relevant query
- The `docs_content` string passed to the model contains both conflicting numbers

## Dependencies

New test-only dependencies (added to `pyproject.toml` dev group):
- `pytest`
- `reportlab`

`chromadb` is already a transitive runtime dependency; `EphemeralClient` is used directly.

## Out of Scope

- FastAPI endpoint testing (`TestClient`)
- Actual LLM output quality (model never called)
- Guardrails extraction (`conversation/guards.py`)
- Performance or latency benchmarks
- Fixes to existing code (characterization only)

## pytest Config

Add to `pyproject.toml`:

```toml
[tool.pytest.ini_options]
testpaths = ["tests"]
markers = ["adversarial: adversarial document robustness tests"]
```
